# 快速开始

欢迎使用ETL-GO！本指南将帮助您快速了解和使用ETL-GO进行数据集成和ETL任务管理。

## 什么是ETL-GO？

ETL-GO是一款开源的数据集成工具，专注于数据抽取（Extract）、转换（Transform）、加载（Load）过程的自动化和简化。它提供可视化配置界面，让您能够快速创建和管理复杂的数据处理任务。

### 核心特点

- **开箱即用**：无需复杂配置，下载即可使用
- **可视化界面**：Web界面进行任务配置和监控
- **多数据源支持**：MySQL、PostgreSQL、SQLite、Doris、CSV、JSON等
- **插件化架构**：易于扩展，支持自定义组件
- **任务调度**：支持定时任务和手动触发
- **文件管理**：内置文件上传和管理功能

## 系统要求

### 硬件要求
- **内存**: 至少512MB RAM（推荐1GB）
- **存储**: 至少100MB可用磁盘空间
- **网络**: 需要访问数据源和目标的网络连接

### 软件要求
- **操作系统**: Windows, Linux, macOS
- **运行环境**: Go 1.18+（仅编译时需要）
- **浏览器**: Chrome 80+, Firefox 75+, Safari 13+

### 数据源要求
- MySQL 5.7+
- PostgreSQL 10+
- SQLite 3.x
- Doris 1.x

## 安装方式

### 方式一：使用预编译包（推荐）

1. **下载最新版本**
   - 访问 [GitHub Releases](https://github.com/BernardSimon/etl-go/releases)
   - 下载对应您操作系统的预编译包

2. **不同系统的安装步骤**

**Linux/macOS:**
```bash
# 下载
wget https://github.com/BernardSimon/etl-go/releases/download/v1.0.0/etl-go-linux-amd64.tar.gz

# 解压
tar -zxvf etl-go-linux-amd64.tar.gz

# 进入目录
cd etl-go-linux-amd64

# 运行
./etl-go
```

**Windows:**
1. 下载 `etl-go-windows-amd64.zip`
2. 解压到任意目录
3. 双击运行 `etl-go.exe`

### 方式二：从源码编译

如果您需要特定功能或自定义修改，可以从源码编译：

```bash
# 克隆代码库
git clone https://github.com/BernardSimon/etl-go.git
cd etl-go

# 安装依赖
go mod download

# 构建
go build -o etl-go main.go

# 运行
./etl-go
```

### 方式三：Docker运行

```bash
# 拉取镜像
docker pull bernardsimon/etl-go:latest

# 运行容器
docker run -d \
  --name etl-go \
  -p 8081:8081 \
  -v /path/to/config:/app/config \
  -v /path/to/data:/app/data \
  bernardsimon/etl-go:latest
```

## 基本概念

### 任务（Task）
任务是ETL-GO中的基本执行单元，包含以下组成部分：

1. **数据源（Source）**：数据输入，如数据库查询、文件读取
2. **处理器（Processor）**：数据转换和处理逻辑
3. **数据目标（Sink）**：数据输出位置，如数据库表、文件
4. **调度配置（Schedule）**：任务执行时间规则

### 组件类型

| 组件类型 | 描述 | 示例 |
|---------|------|------|
| **数据源** | 数据输入组件 | MySQL查询、CSV文件读取 |
| **处理器** | 数据转换组件 | 类型转换、数据过滤、列重命名 |
| **数据目标** | 数据输出组件 | 数据库写入、文件输出 |
| **执行器** | 执行SQL语句 | 数据清理、表创建 |

### 变量系统
支持动态变量，可以在任务配置中使用：
- **系统变量**：全局可用变量
- **SQL变量**：通过SQL查询获取的变量
- **任务变量**：任务执行过程中产生的变量

## 创建第一个任务

让我们创建一个简单的任务，从CSV文件读取数据，处理后写入数据库。

### 步骤1：上传数据文件

1. 登录ETL-GO管理界面
2. 点击左侧菜单"文件管理"
3. 点击"上传文件"按钮
4. 选择您的CSV文件（示例内容如下）：

**users.csv:**
```csv
id,name,email,status,created_at
1,张三,zhangsan@example.com,active,2024-01-01
2,李四,lisi@example.com,inactive,2024-01-02
3,王五,wangwu@example.com,active,2024-01-03
```

### 步骤2：创建任务

1. 点击左侧菜单"任务管理"
2. 点击"新建任务"按钮
3. 填写任务信息：

```yaml
任务名称: "用户数据导入"
任务描述: "从CSV文件导入用户数据到数据库"
调度方式: "手动执行"
```

### 步骤3：配置数据源

在任务配置中，设置数据源：

```yaml
数据源类型: "CSV文件"
文件路径: "选择刚才上传的CSV文件"
编码格式: "UTF-8"
分隔符: ","
包含表头: "是"
```

### 步骤4：添加处理器

添加数据处理步骤：

1. **数据类型转换**：
   ```yaml
   处理器类型: "convertType"
   转换规则:
     - 字段: "id", 类型: "整数"
     - 字段: "created_at", 类型: "日期时间", 格式: "yyyy-MM-dd"
   ```

2. **数据过滤**：
   ```yaml
   处理器类型: "filterRows"
   过滤条件: "status = 'active'"
   ```

### 步骤5：配置数据目标

设置数据输出：

```yaml
目标类型: "SQL表"
数据库连接: "选择您的MySQL连接"
表名: "users"
写入模式: "插入"
批处理大小: "1000"
```

### 步骤6：保存并执行

1. 点击"保存"按钮
2. 返回任务列表
3. 找到刚创建的任务，点击"执行"按钮
4. 在"任务记录"中查看执行结果

## 配置数据源连接

在开始更多任务前，需要配置数据源连接。

### MySQL连接配置

1. 点击左侧菜单"数据源管理"
2. 点击"新建数据源"
3. 填写连接信息：

```yaml
数据源类型: "MySQL"
连接名称: "生产数据库"
主机地址: "localhost"
端口: "3306"
数据库名: "mydb"
用户名: "root"
密码: "********"
```

### 测试连接
配置完成后，点击"测试连接"验证配置是否正确。

## 任务调度

ETL-GO支持多种调度方式：

### 定时调度
使用Cron表达式配置定期执行：
```yaml
调度方式: "定时调度"
Cron表达式: "0 2 * * *"  # 每天凌晨2点执行
```

### 间隔调度
固定时间间隔执行：
```yaml
调度方式: "间隔调度"
间隔时间: "1h"  # 每小时执行一次
```

### 手动执行
仅当手动触发时执行：
```yaml
调度方式: "手动执行"
```

## 故障排除

### 常见问题

#### 1. 服务无法启动
**可能原因**：
- 端口被占用
- 配置文件错误
- 数据库连接失败

**解决方法**：
```bash
# 检查端口占用
netstat -tlnp | grep 8081

# 检查配置文件
cat config.yaml

# 检查日志
tail -f log/app.log
```

#### 2. 任务执行失败
**可能原因**：
- 数据源连接失败
- SQL语句错误
- 文件权限问题

**解决方法**：
1. 检查数据源连接配置
2. 查看任务执行日志
3. 验证文件路径和权限

#### 3. 性能问题
**可能原因**：
- 数据量过大
- 网络延迟
- 数据库性能瓶颈

**优化建议**：
1. 启用批处理
2. 增加超时时间
3. 优化SQL查询
