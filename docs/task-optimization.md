
# 任务优化

ETL-GO 的执行效率与任务配置、数据源性能、并发参数有关。

## 关键配置项

- `pipeline.batchSize`：批量写入大小
- `pipeline.channelSize`：内部通道缓存大小
- 数据源连接池设置：`maxOpenConns` / `maxIdleConns`

## 调优建议

### 1. `batchSize`

- 太小：吞吐低，写入频繁。
- 太大：单批失败成本高、内存压力大。
- 建议：从 `200~1000` 起步，按目标库能力逐步调优。

### 2. `channelSize`

- 太小：上下游容易互相阻塞。
- 太大：内存占用明显上升。
- 建议：先用默认值，出现积压再按阶段问题调整。

### 3. Processor 链复杂度

- 尽量先 `selectColumns` 降字段，再做转换和脱敏。
- 避免在 Processor 中做重型计算。

### 4. Source/Sink 策略

- Source 侧分页或分片读取大表。
- Sink 侧确保目标库索引和写入策略合理。

## 优化排查清单

1. 任务慢在“读取”还是“写入”？
2. 是否存在单个 Processor 成为瓶颈？
3. 数据源连接池是否打满？
4. 是否频繁重试导致实际吞吐下降？

## 常见场景

- 批量写入 CSV/JSON 文件时，适当提高批量大小
- 多个数据库并发访问时，增加连接池容量
- 非关系数据清洗可在 Processor 里尽量简化逻辑

## 常见问题

### 1. 吞吐突然下降

- 检查目标库锁等待、慢 SQL、网络抖动。

### 2. 内存持续升高

- 降低 `channelSize` 和 `batchSize`，并检查是否有超大记录。
