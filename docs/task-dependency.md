# 任务依赖

任务依赖是ETL工作流中的核心概念，用于定义任务之间的执行顺序和依赖关系。ETL-GO提供了灵活的任务依赖管理机制，支持多种依赖类型和复杂的工作流编排。

## 依赖类型

### 1. 执行顺序依赖

#### 线性依赖（顺序执行）
```yaml
线性依赖示例:
  任务A → 任务B → 任务C:
    - 依赖关系: 顺序依赖
    - 执行方式: 串行执行
    - 触发条件: 前序任务成功完成
    - 失败处理: 后续任务不执行
    
  配置示例:
    - 任务A: 数据抽取
    - 任务B: 数据清洗（依赖任务A）
    - 任务C: 数据加载（依赖任务B）
```

#### 并行依赖（并行执行）
```yaml
并行依赖示例:
  任务A → 任务B
        → 任务C:
    - 依赖关系: 并行依赖
    - 执行方式: 并行执行
    - 触发条件: 前序任务成功完成
    - 失败处理: 独立处理
    
  配置示例:
    - 任务A: 主数据准备
    - 任务B: 维度表处理（与任务C并行）
    - 任务C: 事实表处理（与任务B并行）
```

### 2. 数据依赖

#### 文件依赖
```yaml
文件依赖:
  依赖类型:
    - 文件存在依赖: 等待文件生成
    - 文件内容依赖: 检查文件内容
    - 文件大小依赖: 验证文件大小
    - 文件时间依赖: 检查文件时间戳
  
  配置示例:
    - 前置条件: 等待 /data/input/file.csv 生成
    - 检查条件: 文件大小 > 1MB
    - 超时设置: 最长等待30分钟
    - 重试策略: 每5分钟检查一次
```

#### 数据库依赖
```yaml
数据库依赖:
  依赖类型:
    - 表存在依赖: 检查表是否存在
    - 数据量依赖: 验证数据记录数
    - 数据质量依赖: 检查数据完整性
    - 时间戳依赖: 验证最后更新时间
  
  配置示例:
    - 前置条件: users表记录数 > 10000
    - 检查SQL: SELECT COUNT(*) FROM users
    - 超时设置: 最长等待1小时
    - 重试间隔: 每10分钟检查一次
```

### 3. 时间依赖

#### 时间窗口依赖
```yaml
时间窗口依赖:
  依赖类型:
    - 固定时间: 每天9:00执行
    - 时间间隔: 每2小时执行一次
    - 业务时间: 每月1号执行
    - 相对时间: 前序任务完成后30分钟
  
  配置示例:
    - 触发时间: 每天 02:00 (业务低峰期)
    - 时间窗口: 执行时间不超过4小时
    - 超时处理: 超时后强制终止
    - 节假日排除: 节假日不执行
```

#### 日历依赖
```yaml
日历依赖:
  依赖类型:
    - 工作日依赖: 仅工作日执行
    - 节假日依赖: 节假日不执行
    - 特殊日期: 特定日期执行
    - 时间排除: 排除特定时间段
  
  配置示例:
    - 执行日历: 周一至周五
    - 排除日期: 国家法定节假日
    - 时间段: 02:00-06:00
    - 特殊规则: 月末最后一天额外执行
```

```yaml
    - 重试次数: 3

  条件依赖:
    - 父任务: "task_001"
    - 条件类型: "output_condition"
    - 条件表达式: "output.records > 1000"
    - 评估时间: "immediate"
```

```yaml
    - 超时控制: "全局超时2h"
```

## 故障排除

### 1. 常见问题处理

#### 依赖相关问题
```yaml
依赖问题诊断:
  依赖不满足:
    症状: "任务等待依赖超时"
    可能原因:
      - 前序任务失败
      - 依赖条件不满足
      - 网络/资源问题
      - 配置错误
    解决步骤:
      1. 检查前序任务状态
      2. 验证依赖条件
      3. 检查网络和资源
      4. 检查配置
  
  循环依赖:
    症状: "任务无法开始执行"
    可能原因:
      - 直接循环依赖
      - 间接循环依赖
      - 配置错误
      - 动态生成依赖
    解决步骤:
      1. 运行依赖环检测
      2. 分析依赖图
      3. 重构依赖关系
      4. 添加环检测机制
  
  依赖冲突:
    症状: "多个任务竞争同一依赖"
    可能原因:
      - 资源共享冲突
      - 时间窗口重叠
      - 优先级冲突
      - 资源限制
    解决步骤:
      1. 分析资源使用
      2. 调整执行时间
      3. 设置优先级
      4. 增加资源
```

#### 性能问题处理
```yaml
性能问题处理:
  依赖解析慢:
    症状: "依赖解析耗时过长"
    可能原因:
      - 依赖数量过多
```
