
# 任务依赖

当前版本的 ETL-GO 侧重任务执行与调度管理，暂不提供复杂的任务依赖链路。

## 现状说明

- 支持任务模板复用
- 支持手动和定时执行
- 目前没有内置跨任务依赖关系调度

## 可行方案

如果需要实现任务依赖，可通过以下方式模拟：

- 在任务 A 成功后，由外部调度器触发任务 B 的运行 API
- 使用任务记录表或业务状态表作为“依赖信号”
- 通过第三方调度系统（Airflow/xxl-job/Jenkins）编排调用顺序

## 推荐实现模式

1. 把 ETL-GO 作为执行引擎，依赖关系交给外部编排。
2. 以“任务记录状态=成功”作为下游触发条件。
3. 增加重试和告警，避免单点失败导致链路中断。

## 注意事项

- 不建议在单任务内塞入过多跨业务步骤来模拟依赖。
- 依赖链越长，越需要明确超时、重试和补偿策略。
