
# 日志分析

任务日志帮助定位失败原因、执行耗时和数据错误。

## 日志查看方式

1. 打开任务记录页面
2. 选择某次执行记录
3. 点击“查看日志”

## 日志来源

- 任务运行日志：按任务记录聚合，定位单次执行问题
- 系统日志：`config.log.filename`（默认 `./log/app.log`）

## 常见排查方向

- SQL 语法错误
- 数据类型转换失败
- 输出目标连接异常
- 网络 / HTTP 请求超时

## 推荐排查顺序

1. 定位失败组件（Source/Processor/Sink/Executor）。
2. 查看报错前最后一条成功日志。
3. 对照任务参数快照确认是否变量替换异常。
4. 必要时降低数据量做最小复现。

## 优化建议

- 尽量将错误信息写入任务日志
- 通过 `filterRows` 和 `convertType` 提前验证数据
- 对 HTTP 源和 Sink 设置合理超时时间

## 常见问题

### 1. 日志为空或很少

- 任务可能在装配阶段失败，查看任务记录 message 与系统日志。

### 2. 只有 `context canceled`

- 某个上游组件先失败导致全链路取消，需找最早的真实错误。
