# 查看任务执行情况

任务执行情况查看是监控和管理 ETL 任务的重要功能。ETL-GO 提供了完整的任务执行记录查看、状态监控、结果分析和故障诊断功能，帮助您全面了解任务的执行状况。

## 执行记录概述

### 记录类型

| 记录类型 | 描述 | 包含信息 |
|----------|------|----------|
| **调度执行记录** | 定时调度触发的执行 | 调度时间、执行状态、耗时、结果 |
| **手动执行记录** | 用户手动触发的执行 | 触发用户、执行时间、状态、结果 |
| **立即执行记录** | API 触发的立即执行 | API调用信息、执行状态、结果 |
| **测试执行记录** | 测试环境的执行 | 测试配置、执行环境、验证结果 |

### 执行状态

| 状态码 | 状态 | 描述 | 颜色标识 |
|--------|------|------|----------|
| **0** | 运行中 | 任务正在执行 | 🟡 黄色 |
| **1** | 运行成功 | 任务执行成功完成 | 🟢 绿色 |
| **2** | 运行失败 | 任务执行失败 | 🔴 红色 |
| **3** | 已取消 | 任务被手动取消 | ⚪ 灰色 |
| **4** | 超时 | 任务执行超时 | 🟠 橙色 |
| **5** | 部分成功 | 任务部分成功 | 🔵 蓝色 |

- task_id: 任务ID (可选)
- status: 执行状态 (可选，0-5)

- end_time: 结束时间 (可选，ISO格式)
- page: 页码 (默认1)
- page_size: 每页数量 (默认20)
- sort_by: 排序字段 (可选，如 start_time)

## 注意事项

1. **记录完整性**: 确保重要执行记录完整保存
2. **性能影响**: 大量记录可能影响查询性能
3. **存储空间**: 记录占用存储空间，需合理配置保留策略
4. **隐私安全**: 敏感数据需脱敏处理
5. **查询性能**: 大时间范围查询需分页处理
6. **数据一致性**: 确保记录数据与执行结果一致
7. **版本兼容**: 系统升级需考虑记录格式兼容性
8. **备份恢复**: 重要记录需定期备份

## 故障排除

### 常见问题

#### 1. 记录查看缓慢
**问题**: 查看执行记录响应缓慢
**可能原因**:
- 记录数量过多
- 查询条件复杂
- 数据库性能问题
- 网络延迟

**解决方案**:
```yaml
优化措施:
  1. 添加分页查询
  2. 优化查询条件
  3. 建立数据库索引
  4. 增加查询缓存
  5. 使用异步查询
```

#### 2. 记录数据不一致
**问题**: 执行记录与实际情况不一致
**可能原因**:
- 记录写入延迟
- 数据同步问题
- 程序缺陷
- 系统异常

**解决方案**:
```yaml
检查步骤:
  1. 验证记录写入逻辑
  2. 检查数据同步机制
  3. 审核程序代码
  4. 检查系统日志
  5. 数据一致性校验
```

#### 3. 记录丢失
**问题**: 部分执行记录丢失
**可能原因**:
- 记录清理过早
- 写入失败
- 存储故障
- 程序异常

**解决方案**:
```yaml
预防措施:
  1. 调整保留策略
  2. 增加写入确认
  3. 实施备份机制
  4. 添加监控告警
  5. 定期完整性检查
```
