
# 查看任务执行情况

任务执行记录包含运行状态、开始结束时间、消息和输出文件。

## 记录内容

- 任务名称
- 执行状态：`running` / `success` / `failed`
- 开始时间 / 结束时间
- 运行日志摘录
- 关联输出文件

状态码说明：

- `0`：运行中
- `1`：执行成功
- `2`：执行失败或被中止

## 常用操作

- 查看执行详情
- 取消正在运行的任务
- 下载任务输出文件
- 查看运行参数

## 建议排查路径

1. 先看记录状态与 message。
2. 再看该记录的参数快照（确认变量替换结果）。
3. 最后看详细日志定位组件级错误。

## 常见问题

### 1. 记录状态一直是运行中

- 可能是 Source/HTTP 请求未结束。
- 可尝试手动取消并检查日志最后停留位置。

### 2. 状态失败但 message 不够详细

- 打开任务日志详情查看完整错误栈。

### 3. 取消后仍显示运行中

- 取消是异步生效，等待刷新状态后确认。
