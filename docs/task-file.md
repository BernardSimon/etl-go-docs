
# 文件管理

ETL-GO 将文件视为平台资产，可在任务配置中复用文件资源。

## 文件上传

进入文件管理页面，可上传 CSV / JSON 等任务文件。

## 文件使用

在任务配置中，文件型参数可选择已上传的文件，无需手动输入文件 ID。

典型参数：

- `file_id`：单文件输入
- `file_ids`：多文件输入

## 文件类型

- CSV 文件
- JSON 文件
- Sqlite 数据库文件
- 其他文本文件

## 运行期行为

- 文件型参数会在任务运行前解析为 `file_path`。
- Source/Sink 实际读取的是解析后的文件绝对路径。
- 输出文件会关联到任务记录，便于下载与审计。

## 使用建议

- 上传数据文件后优先测试 Source 是否正确解析
- 通过文件资产减少重复上传和路径管理成本

## 常见错误排查

### 1. `config is missing required key 'file_path'`

- `file_id` 无效或文件已删除，导致运行时无法解析。

### 2. CSV/JSON Source 读取失败

- 文件内容格式不合法（列数不一致、JSON 顶层非数组等）。

### 3. 输出文件找不到

- 检查任务记录是否执行失败，失败任务可能只保留错误上下文。
